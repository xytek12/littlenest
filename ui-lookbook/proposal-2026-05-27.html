Tastings · Dark · HEDirect fix for screenshot 5. "פרק: ביסים ראשונים", whisper card, and every allergen label are bright cream against deep plum — fully readable.

What this revision changes from v1

  • Backgrounds — gendered pastel washes (sky-blue for boys, blush-rose for girls, diagonal blend for boy+girl twins) replace the flat cream. Dark mode is now a layered jewel-tone canvas (plum + midnight + teal glows) instead of flat indigo.
  • Typography — Cormorant Garamond italic + Fraunces for English; Frank Ruhl Libre + Heebo for Hebrew so שינה / פרק render with proper serif weight, not a generic system font.
  • Dates — all "yesterday" / "2 hours ago" strings replaced with explicit date+time (e.g. Wed, May 27 · 19:14 or יום שלישי, 26.5 · 14:20–15:45).
  • Feed quick-add — Solids removed. Sheet shows Nursing + Bottle only.
  • Allergen page — no "+" button on any card. Banner is now compact (just a category label). Every allergen is pre-listed and tappable to step through 1/2/3.
  • Dark text — body lifts from #A8A2C9 (low contrast) to #FBF1DC warm cream + #D9C8B6 muted taupe. Banner text on jewel banners is white-cream, not washed pastel.

What stays exactly the same

  • Bottom dock — same five tabs, same emoji icons, active pill follows child accent (boy blue / girl pink / twins gradient / dark cream).
  • Allergen 1/2/3 step flow — same data model, same tap target, just no "+" because allergens are catalog-driven.
  • Sleep / nursing / bottle state machine in PrototypeState.tsx — untouched.
  • Sleep timer on Home — appears when activeSleepStartedAt is set, with pulse + mm:ss + Stop.
  • All existing i18n keys in en.ts / he.ts — preserved; new strings added alongside.

Backend changes (parallel agents)

  • Bump GEMINI_MODEL default in aiProviders.ts from gemini-2.0-flash → gemini-2.5-flash (matches your new Supabase secret).
  • Validate new Gemini key end-to-end against ai-router + recipe-search.
  • Add structured error logging in recipe-search so future 500s tell us what failed (parse vs. fetch vs. domain filter).
